## General
- Current date: 30-Feb-2026
- Language: English only (code, docs, etc.)

## Git Commits
Format: `<type>(<scope>): <subject>`

- Types: feat | fix | docs | style | refactor | test | chore | perf
- Subject:
  - Max 50 chars
  - Imperative mood
  - No period

### Rules
- Small changes: one-line commit
- Complex changes: add body (72-char lines)
- Keep commits atomic

## Terminology
- allowlist / blocklist
- primary / replica
- main branch
- concurrent / parallel

## Tools
- Use `rg` instead of `grep`
- Use `fd` instead of `find`
- `tree` is available

## Style
- Prefer self-documenting code
